### Exported packages.txt mismatch

- `copy_files.sh` exported `packages.txt` with `qlist -Iv`
- That only writes installed package atoms and discards size/build-time metadata
- The root image already contained a different `/packages.txt`, but its size parsing was wrong for entries such as `non-files,0` and its build times were human-readable only

Conclusion: setup 03 was overwriting the useful report with a plain package list.

### Why E14 squashfs is larger

- `gentoo.squashfs` is created with a very aggressive exclusion list:
  - `bin`, `sbin`, `lib`, `lib64`
  - `usr/bin`, `usr/sbin`, `usr/lib`, `usr/lib64`
  - `opt`
  - many caches, logs, docs, locale data
- `gentoo.squashfs_e14` does not exclude those core runtime trees

Conclusion: `gentoo.squashfs_e14` is expected to be much larger. It is not a smaller variant of the base squashfs; it is effectively the fuller runtime image with some hardware-specific cleanup.

### Firefox in E14 squashfs

- Checked inside the final image and by listing `gentoo.squashfs_e14`
- Result: Firefox is still present under `/opt/firefox`
- Result: launcher remains at `/usr/bin/firefox-bin`
- The previous cleanup command only removed paths like `/usr/lib64/firefox` and `/usr/bin/firefox`, which do not cover the `firefox-bin` layout

Conclusion: the old Dockerfile comment and command were misleading. Firefox was never actually removed from the E14 squashfs.
