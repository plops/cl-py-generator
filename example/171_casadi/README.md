## 🔍 Detailed Differences Between Implementations

### 🚗 Active Suspension Series (`gen08` vs `gen09`)

Both programs implement MPC control for a Quarter-Car Model, but they differ fundamentally in how the mathematical optimization problem is represented in memory:

*   **[gen08.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen08.lisp) (Unrolled Loop Formulation):**
    This program creates the state and control sequences as flat lists of independent variables (`x_0`, ..., `x_N`). The dynamic residual constraints and stage costs are constructed using standard Python/Lisp `for` loops that iterate over the horizon $N$.
    *   *Issue:* The size of the symbolic expression graph scales linearly as $O(N)$. For larger horizons, compiling the problem with JIT compilers results in huge C files (often exceeding 10MB) and very long compilation times.
*   **[gen09.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen09.lisp) (Map-based Vector Formulation):**
    This program rewrites the formulation by declaring states and inputs as compact symbolic matrices (`X` and `U`). Instead of loops, it defines single-step dynamics and maps them using CasADi's `.map(N)` and `.mapaccum(N)` methods.
    *   *Benefit:* The symbolic graph size remains constant at $O(1)$ regardless of horizon size. This eliminates Python-level loop overhead, reduces code lines by ~50%, and enables extremely fast JIT compilation times (under 2 seconds instead of 23 seconds).

### 🕹️ Inverted Pendulum Series (`gen11a` to `gen11h`)

This sub-series traces the incremental development of a high-performance control system, beginning with pure simulation and ending with sensor selection, noise injection, state estimation, and model predictive control:

#### 1. Simulation and Solver Benchmarking
*   **[gen11a.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen11a.lisp) (Base Simulator):** Runs a bare-bones ODE simulation of cart-pole dynamics using CasADi's integrator interfaces. It does not contain any feedback or controllers.
*   **[gen11b.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen11b.lisp) (MPC JIT/Warmstart Benchmark):** Implements a basic closed-loop MPC control using IPOPT. It compiles with and without JIT (`gcc -O3`) to verify real-time feasibility. It concludes that warm-starting from the previous solve time step is essential to drop solve times to ~20ms, making JIT optional for 30Hz GUI animations.

#### 2. Interactive Graphical Dashboards
*   **[gen11c.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen11c.lisp) (Mapped JIT Cache GUI):** Wraps the fast MPC loop inside a PySide6 GUI with PyQtGraph visualization. It renders a cart-pole drawing where users can drag slider controls to apply disturbances (wind, weight changes) in real-time.
*   **[gen11f.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen11f.lisp) (Structurally Identical Variant):** Identical to `gen11c.lisp` except for minor window titles and class definitions, serving as a regression test candidate.

#### 3. Solver Parameter Tuning & Optimization Benchmarks
*   **[gen11d.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen11d.lisp) (Headless Controller Tuner):** Executes simulation benchmarks without launching a GUI. It automatically writes out tables comparing IAE (Integral Absolute Error), overshoot, and settling times for 8 different Q/R weight sets, prediction horizons ($N$), and physical parameter mismatches.
*   **[gen11e.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen11e.lisp) (Optimization Parameter Benchmark):** Tests 9 combinations of solvers (primal vs. dual warmstarts, Python vs JIT, `to_function` compiling, and mapped vs unmapped constraints) across small ($N=20$) and large ($N=200$) horizons to establish ultimate speedups (confirming that combining `to_function` + JIT + mapped constraints yields sub-millisecond execution).

#### 4. State Estimation and Noise Injection
*   **[gen11g.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen11g.lisp) (Basic MHE + MPC):** Introduces **MHE (Moving Horizon Estimation)**. Instead of assuming the controller has direct access to noise-free states (Full State Feedback), it injects gaussian noise into the sensors. The MHE solves a tracking optimization problem over a sliding history window ($N_{mhe}=15$) to filter out noise, feeding clean estimates to the MPC.
*   **[gen11h.lisp](file:///home/kiel/stage/cl-py-generator/example/171_casadi/gen11h.lisp) (Advanced MHE + MPC GUI):** Adds dynamic rebuilding capabilities and interactive sensor selection. Users can uncheck individual variables (e.g. measure only position $s$ and angle $\theta$ and hide velocities $v$ and $\omega$). MHE dynamically reconstructs the missing unmeasured velocities from physical constraints and remaining sensor data. It also allows selecting solver methods (L-BFGS vs. exact Hessian) and modifying weights dynamically via GUI sliders.
